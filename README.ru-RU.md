## Что такое Redeven?

Redeven — это единый бинарный файл, который объединяет ваши компьютеры и серверы в одной вкладке браузера. Вместо постоянного переключения между SSH-терминалами, файловыми менеджерами, панелями мониторинга, перенаправлением портов и окнами IDE вы получаете единое рабочее пространство на оборудовании, которое уже контролируете.

Redeven работает на вашем компьютере, удалённых серверах или любом доступном SSH-хосте. Файлы, процессы, ключи API и учётные данные остаются там, где им положено быть: Redeven не передаёт ваши данные в открытом виде через чужую инфраструктуру.

- **Клиенты подключаются к среде выполнения конечной точки**: браузер, Desktop, CLI и сеансы через SSH входят в одно рабочее пространство под управлением среды выполнения.
- **Среда выполнения является границей доверия**: единый бинарный файл Go управляет файлами, терминалами, мониторингом, Git, перенаправлением веб-сервисов, компоновкой Workbench, заметками, настройкой Browser Editor, Flower и доступом к мосту Codex.
- **Передача данных и политики остаются явными**: Flowersec передаёт зашифрованный трафик RPC и потоков, а разрешения сеанса, локальная политика прав, область файловой системы и локальные секреты ограничивают возможности каждого сеанса.

## Быстрый старт

Начать работу можно двумя способами: через Desktop, рекомендуемый большинству пользователей, или через CLI.

<!-- readme-section:desktop-app -->
<a id="desktop-app"></a>

### Приложение Desktop

1. Скачайте Redeven Desktop со страницы [GitHub Releases](https://github.com/floegence/redeven/releases).
2. Откройте приложение и выберите окружение: локальное, Provider, SSH-хост или сохранённый URL.
3. Приступайте к работе: рабочее пространство автоматически откроется в браузере.

Для удалённых компьютеров Desktop может автоматически установить подходящий релиз Redeven через SSH, а затем по вашему выбору явно подключить эту управляемую среду выполнения SSH к окружению типа provider. Ручная настройка удалённого хоста не требуется.

<!-- readme-section:cli -->
<a id="cli"></a>

### CLI

```bash
# 1. Установить
curl -fsSL https://raw.githubusercontent.com/floegence/redeven/main/scripts/install.sh | sh

# 2. Выполнить первичную настройку, прочитав билет из защищённого файла секрета
redeven bootstrap \
  --provider-origin https://<your-provider> \
  --controlplane https://<your-access-point> \
  --env-id <env_public_id> \
  --bootstrap-ticket-file /run/secrets/redeven-bootstrap-ticket

# 3. Запустить
redeven run --mode hybrid

# 4. Открыть http://localhost:23998 в браузере.
```

Первичная настройка записывает несекретные метаданные локального окружения в `~/.redeven/local-environment/config.json`, а учётные данные среды выполнения — в защищённый правами файл `~/.redeven/local-environment/secrets.json`. Каждый пользователь операционной системы имеет одну идентичность локального окружения, которая может быть привязана только к одному окружению типа provider за раз. Desktop и браузер используют один контракт одноразового билета.

Для интерактивной одноразовой настройки замените параметр `--bootstrap-ticket-file ...` на `--bootstrap-ticket-stdin`. Redeven запросит билет, не отображая его. Если stdin является каналом или перенаправлением, тот же параметр прочитает билет напрямую без вывода приглашения.

Системы автоматизации, менеджеры секретов, агенты CI и оркестраторы контейнеров могут напрямую передавать `REDEVEN_BOOTSTRAP_TICKET` и `REDEVEN_LOCAL_UI_PASSWORD`. Переменные окружения остаются доступными для наблюдения процессам того же пользователя, отладчикам и платформе хоста. Для интерактивного использования предпочтительны скрытый ввод, stdin или защищённые файлы секретов.

Интерактивные терминалы по умолчанию используют расширенное представление среды выполнения Redeven: компактный анимированный символьный знак, сведения о версии и протоколе, количество активных сеансов и рабочих нагрузок, URL Local UI и окружения, просмотр журнала среды выполнения в реальном времени, а также панели предупреждений и ошибок с доступными действиями. Используйте клавиши со стрелками для перехода между плоскостью управления, сеансами и журналами. Enter в разделе сеансов открывает фильтруемый список активных сеансов, в журналах — полный журнал среды выполнения, а в плоскости управления — подключение, отключение или поля первичной настройки. Esc возвращает назад, Ctrl+C останавливает среду выполнения. Неинтерактивные оболочки используют обычный текст. Запуски под управлением Desktop вместо терминального интерфейса используют машинный контракт представления со структурированными отчётами о запуске.

**Кратко о режимах запуска:**

| Цель | Команда |
|---|---|
| Только Local UI на этом устройстве | `redeven run --mode local` |
| Local UI и канал удалённого управления | `redeven run --mode hybrid` |
| Среда выполнения под управлением Desktop | `redeven run --mode desktop --desktop-managed --presentation machine --local-ui-bind localhost:23998` |
| Доступ с другого устройства | Оставьте Local UI на loopback-интерфейсе и используйте Redeven Desktop, перенаправление SSH или защищённый туннель Flowersec. |

## Возможности

| Интерфейс | Что он предоставляет |
|---|---|
| Файлы и Git | Загрузка и скачивание файлов, встроенный просмотр и редактирование, изменения Git в пределах папки, сравнение различий и рабочие процессы stash. |
| Терминал | Терминалы с несколькими вкладками, открытые в рабочих каталогах и использующие ту же модель разрешений среды выполнения. |
| Мониторинг | Представления процессора, памяти, диска, сети и процессов из среды выполнения конечной точки. |
| Browser Editor | Сеансы редактора в браузере, явно настроенные Desktop и изолированные по рабочим областям. |
| Веб-сервисы | Регистрация сервисов и доступ через перенаправление портов под управлением среды выполнения, без ручной настройки SSH-туннелей. |
| Flower и Codex | Необязательные интерфейсы ИИ, использующие проверенные средой выполнения инструменты и локальную конфигурацию модели и хоста. |
| Desktop | Нативное средство запуска для локальных, размещённых на стороне provider, настроенных через SSH и сохранённых окружений Local UI. |

## Безопасность, которая не заслоняет возможности

Redeven делает акцент на возможностях, однако среда выполнения остаётся границей доверия, поскольку контролирует реальный хост.

- Среда выполнения находится на конечной точке и сохраняет там данные в открытом виде.
- Плоскость управления выдаёт данные первичной настройки, разрешения и неизменяемые метаданные сеанса.
- [Flowersec](https://github.com/floegence/flowersec) передаёт зашифрованные байты между клиентом и средой выполнения конечной точки. Текущая интеграция документирована для `flowersec-go/v0.20.2`.
- Эффективные разрешения формируются из выданных сервером разрешений сеанса и ограничиваются локальной политикой (`read`, `write`, `execute`, `admin`; ни одна категория не подразумевает другую).
- Локальная конфигурация, материалы E2EE, журналы аудита и диагностика остаются в каталоге состояния конечной точки.
- GitHub Releases остаётся общедоступным источником истины для бинарных файлов, контрольных сумм, подписей и ресурсов проверки OKF.

## Документация

Redeven хранит поддерживаемые знания репозитория в [OKF v0.1](okf/index.md). Корпус OKF формируется из актуального поведения исходного кода и встраивается в среду выполнения для `okf.search`.

Машиночитаемый интерфейс интеграции с provider находится в [spec/openapi/rcpp-v2.yaml](spec/openapi/rcpp-v2.yaml). За пределами OKF поддерживаемые файлы Markdown намеренно ограничены `AGENTS.md`, `THIRD_PARTY_NOTICES.md`, каноническим `README.md` и поддерживаемыми переводами `README.<locale>.md`, объявленными в `assets/readme/locales.json`.

go build -o redeven ./cmd/redeven

<!-- readme-section:local-guardrails -->
<a id="local-guardrails"></a>

### Локальные защитные проверки

```bash
./scripts/install_git_hooks.sh
node scripts/generate_third_party_notices.mjs --check
```

Примечания:

- Ресурсы `internal/**/dist/` генерируются и встраиваются с помощью Go `embed`.
- Ресурсы `dist` клиентской части не сохраняются в Git. Версионируемое исключение — `okf/dist/*`, которое остаётся в репозитории как проверяемые метаданные релиза пакета OKF.
- `THIRD_PARTY_NOTICES.md` генерируется из модулей Go и lock-файлов JavaScript. После изменения зависимостей запустите `node scripts/generate_third_party_notices.mjs` и убедитесь, что `--check` проходит.
- `./scripts/lint_ui.sh`, `./scripts/check_desktop.sh`, `./scripts/build_assets.sh` и `go test ./...` являются основными проверками исходного кода.
- `./scripts/dev_desktop.sh` запускает Desktop из текущего checkout или worktree со свежей сборкой среды выполнения.
- `cd desktop && npm run start` и `cd desktop && npm run package` подготавливают `desktop/.bundle/<goos>-<goarch>/redeven` перед запуском Electron или упаковкой оболочки Desktop.

</details>

<summary>Локальное состояние, пути релиза и устранение неполадок</summary>

- GitHub Releases является общедоступным источником истины для версионированных архивов CLI, установщиков Desktop, контрольных сумм, подписей и ресурсов проверки OKF.
